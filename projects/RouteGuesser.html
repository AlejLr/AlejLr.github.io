What is RouteGuesser?

A map-based game to help new students learn Leiden's street network by playing with real routes. The browser UI guides a player from a start to an end location while a Flask backend computes the optimal route and validates each move. The result is a compact, production-minded build: a single Docker image, fast startup, and an API-first contract that kept a 6 person team moving in parallel.

  • Purpose: introduce new students to the city through simple game play.
  • Approach: clean JSON communication between a Leaflet UI and Flask endpoints.
  • Outcome: delivered a complete end-to-end solution with strong software quality

Gameplay

  1. Select difficulty and number of rounds.
  2. POST /api/main with { type: "start", difficulty } → backend returns start/end and blocked roads.
  3. Click white node markers to extend your path; UI calls POST /api/main with { type: "neighbours" } to get valid moves.
  4. Reach the destination → score calculated vs the optimal path; animated progress feedback.
  5. Repeat for the chosen number of routes; view final summary.
  • Scoring: score = (optimal_distance / player_distance) x 100% (capped at 100%).
  • Difficulties: Easy (no blocks), Medium (~50 blocked roads), Hard (~100 + non initially known blocked roads).
  • UI: Leaflet map centered on Leiden with start (green), end (blue), blocked roads (red).

API & Architecture

Stack: Flask 2.2.2 (Python 3.11), HTML5, CSS3, Vanilla JS, Leaflet.js, Flask-CORS (local dev), NetworkX, NumPy, GeoJSON, Matplotlib. Deployed via Docker and Google Cloud Run.

Results & what this demonstrates

  • Full-stack delivery: Flask API + responsive JS/Leaflet UI with smooth team collaboration.
  • Clean, versioned JSON contracts that kept client/server decoupled.
  • From local Docker to Google Cloud Run with a minimal, reliable release process.
  • Solid data-structure design (graph routing) with pragmatic performance.
